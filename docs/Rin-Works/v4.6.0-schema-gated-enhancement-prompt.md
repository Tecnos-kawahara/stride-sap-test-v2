## 背景

論文 "Talk Freely, Execute Strictly: Schema-Gated Agentic AI for Flexible and Reproducible Scientific Workflows"（Cook et al., Cambridge, 2026, arXiv:2603.06394）の知見を Tecnos-STRIDE に取り込む。

STRIDE は既にスキーマゲート型パラダイムの実装（phase_gate.py + stride-lint + APPROVAL.md）を持つが、以下が不足している:
1. AIの実行権限スコープが暗黙的（mode_policy.yaml に散在）で、一箇所に宣言されていない
2. 既存3モード（autopilot/confirm/validate）に学術的フレームワークの裏付けがない
3. プロジェクト設計時に「決定性 vs 柔軟性」のトレードオフを意識的に選択する仕組みがない

**学術的引用（改善の理論的根拠）:**
- Cook, S. et al. (2026). "Talk Freely, Execute Strictly: Schema-Gated Agentic AI for Flexible and Reproducible Scientific Workflows." arXiv:2603.06394.
- Key concepts: Execution Authority separation, ED/CF Pareto front, Validation Scope Spectrum

```yaml

# Reference: Cook et al. (2026) "Talk Freely, Execute Strictly" arXiv:2603.06394
# Purpose: AI agent の権限スコープを明示宣言し、監査・ガバナンスを支援する
#
# conversational: AI が自由に実行できる行為（承認不要）
# gated: スキーマ検証（stride-lint PASS）を条件に実行可能な行為
# prohibited: AI に実行権限がない行為（人間専用）
execution_authority:
  conversational:
    description: "AIが承認なしに自由に実行できる行為"
    actions:
      - "仕様の解釈・質問・明確化"
      - "候補アクションの提案"
      - "stride-lint APPROVAL_PENDING以外のエラー自動修正"

      - "テスト実行・結果収集"

    description: "スキーマ検証を通過した場合のみ実行可能な行為"
    gate_mechanism: "stride-lint + phase_gate.py"
    actions:
      - "Phase内の成果物ファイル作成・編集（phase_gate.py ALLOWED時のみ）"
      - "次Phase成果物の着手（前Phase Gate承認後のみ）"

      - "Phase Gateのスキップ"
      - "coverage_tier の事後変更（basic_design承認後）"
```

### Task 4: `wi_readiness_checker.py` に `execution_authority` 参照の検証を追加

**目的:** mode_policy.yaml の execution_authority 宣言に対して、WI の mode が検証スコープ（validation scope）と整合しているか検証する。Check 3（mode_policy）が risk_flags → mode の「ポリシー準拠」を見るのに対し、Check 8 は execution_authority 宣言があるときにのみ発火し、「gated 行為に必要な検証スコープを WI の mode が満たしているか」を確認する。

**⚠️ 重要: コード構造の正確な把握**

以下は実コードから確認した事実:
- フォールバック辞書は `run_readiness_checks()` 関数内のローカル変数 `mode_policy`（577行付近）にインラインで定義されている（定数名ではない）
- `results` は `list[CheckResult]` 型。各チェック関数は `CheckResult` を返す
- `mode_policy` 変数は `run_readiness_checks()` のスコープ内でのみ利用可能
- 既存のチェックは Check 1〜7（gate5, wi_def, mode_policy, pre_run, cross_dep, ops_pack, state）
- セルフテストは 14 件（Test 1〜14）
- 推奨 Mode 判定は `_get_recommended_mode()` 関数（125行付近）が担当し、`risk_flag_mapping` 辞書形式（本番）と `rules` 配列形式（フォールバック）の両方に対応
- `_apply_autonomy_bias()` 関数（156行付近）が bias シフトと `tier_mode_minimum` の適用を担当

**設計方針:**
- Check 8 は `_get_recommended_mode()` と `_apply_autonomy_bias()` を**再利用**する（ハードコードした risk_flags リストを持たない）
- Check 3 との責務分離: Check 3 は「WI の mode がポリシー推奨以上か」、Check 8 は「execution_authority が宣言されている場合に、gated 行為の検証スコープ（推奨 mode）と実際の mode の差分を execution_authority の文脈で報告する」
- execution_authority が宣言されていないポリシー（レガシー）では Check 8 は PASS を返す

#### 4a. 新しいチェック関数 `check_execution_authority()` を追加

`check_state_consistency()` 関数の直後（`run_readiness_checks()` 関数の直前、約550行付近）に以下を追加:

```python
def check_execution_authority(
    fm: dict[str, Any],
    mode_policy: dict[str, Any],
    autonomy_bias: str = "balanced",
    coverage_tier: str = "standard",
) -> CheckResult:
    """Check 8: Validate WI against execution_authority declaration in mode policy.

    When execution_authority is declared, verify that the WI's mode provides
    sufficient validation scope for gated actions. Uses the same policy-driven
    _get_recommended_mode() and _apply_autonomy_bias() as Check 3 — no
    hardcoded risk flag lists.

    Distinct from Check 3:
    - Check 3: "Is mode >= recommended?" (pure policy compliance)
    - Check 8: "Does mode satisfy execution_authority's validation scope
      requirements?" (authority-aware context reporting)
    """
    ea = mode_policy.get("execution_authority")
    if not ea:
        return CheckResult("PASS", "No execution_authority declared in policy (legacy mode)")

    mode = fm.get("mode", "autopilot")
    risk_flags = fm.get("risk_flags") or []
    gate_mechanism = ea.get("gated", {}).get("gate_mechanism", "stride-lint + phase_gate.py")

    # Use the same policy-driven recommendation as Check 3
    recommended = _get_recommended_mode(list(risk_flags), mode_policy)
    recommended = _apply_autonomy_bias(recommended, autonomy_bias, mode_policy, coverage_tier)

    # When execution_authority is declared, the recommended mode represents
    # the minimum validation scope for gated actions
    if MODE_ORDER.get(mode, 0) < MODE_ORDER.get(recommended, 0):
        override = fm.get("mode_override") or {}
        reason = override.get("reason")
        if reason:
            return CheckResult(
                "WARN",
                f"Execution authority: mode='{mode}' below recommended validation scope "
                f"'{recommended}' (override: {reason}, gate: {gate_mechanism})",
            )
        return CheckResult(
            "FAIL",
            f"Execution authority violation: mode='{mode}' does not satisfy "
            f"validation scope '{recommended}' required for gated actions",
            details=[
                f"Gate mechanism: {gate_mechanism}",
                f"Recommended validation scope: {recommended} (from risk_flags + bias + tier)",
                f"Declared gated actions: {len(ea.get('gated', {}).get('actions', []))}",
                f"Declared prohibited actions: {len(ea.get('prohibited', {}).get('actions', []))}",
                "Per Article XIV, gated actions require mode >= recommended validation scope",
            ],
        )

    return CheckResult(
        "PASS",
        f"Execution authority aligned (mode={mode}, scope={recommended}, "
        f"gate={gate_mechanism})",
    )
```

```markdown
AI agentの権限は3層に分離される。詳細は `shared/policies/mode_policy.yaml` の `execution_authority` セクションを参照。

| 権限層 | 説明 | ゲート機構 |
|--------|------|-----------|
| **Conversational** | 解釈・提案・lint修正 | なし（自律実行） |
| **Gated** | 成果物作成・WI着手・PR作成 | stride-lint + phase_gate.py + wi_readiness_checker.py |
| **Prohibited** | APPROVAL.md編集・Gate skip | 絶対禁止（既存ルールで担保済み） |

### 3モードと検証スコープの対応

| Mode | 検証スコープ | 対応する権限層 |
|------|-------------|---------------|
| autopilot | ツールレベル（stride-lint PASS） | conversational + gated（最小検証） |
| confirm | ワークフローレベル（+ plan_review） | conversational + gated（中程度検証） |
| validate | ドメインレベル（+ design_diff_review） | conversational + gated（最大検証） |
```
