How To Use This Template

推奨: 業務レビューではこの HTML を使い、詳細調査は playwright-report/index.html と trace.zip に委譲します。
Fill Targets
  • Run metadata: 実行日時、環境、コミット、対象ブラウザ。
  • Scenario summary: 全 10 ケースの PASS/FAIL/Warning。
  • Business actuals: journal_entry_id、clearing_status、close_report_id。
  • Evidence links: screenshot、payload、trace、video。
  • Review note: 業務判断に必要な差異だけを短く残す。
Best Practice
  • 成功ケースは 2〜3 枚の key-step screenshot に絞る。
  • 異常系は error screen と response payload を必ず並べる。
  • 月次締め系は CSV / PDF / review screen を 1 ケースに束ねる。
  • BPMN ID とシナリオ ID を同じ詳細カード内に出す。
