# テストパターンマスタ定義書 v2

> 本ドキュメントは test_perspective_master.yaml の人間可読リファレンスです。
>
> 対象ファイル: `extensions/sap/config/test_perspective_master.yaml`
>
> AIによるABAP自動開発で使用されるテストパターンのマスタ定義を人間が読める形式で整理したものです。
> 新しい処理種別やテストパターンを追加する場合は、このドキュメントと YAML ファイルの両方を更新してください。

---

## 構造の説明 -- 3階層モデル

このマスタは以下の3階層で構成されています。

```
処理種別 (processing_type: AUTH, EXIST, BAPI 等)
  +-- テストパターン (branches: 正常系/異常系の分岐)
        +-- テスト観点コード (perspectives: BRANCH_BIZ, ERROR_MSG 等)
```

**各階層の役割:**

1. **処理種別 (processing_type)** -- プログラム内の各処理ブロックの種類を表す（例: AUTH = 権限チェック, BAPI = BAPI呼出）
2. **テストパターン (branches)** -- 各処理種別に紐づく正常系・異常系の分岐パターン（例: BAPI -> 「更新成功」「更新失敗」）
3. **テスト観点コード (perspectives)** -- 各テストパターンで何を検証するかを示すコード（例: 「更新成功」-> BRANCH_BIZ, BAPI_RETURN_VERIFY, BAPI_COMMIT_VERIFY）

**具体例（BAPI の場合）:**

| 処理種別 | テストパターン | 正常/異常 | テスト観点コード | 検証内容 |
|---------|-------------|----------|----------------|---------|
| BAPI（BAPI呼出） | 更新成功 | 正常 | BRANCH_BIZ | エラーなしで後続処理に進むこと |
| BAPI（BAPI呼出） | 更新成功 | 正常 | BAPI_RETURN_VERIFY | BAPI RETURN テーブルにエラーが含まれないこと |
| BAPI（BAPI呼出） | 更新成功 | 正常 | BAPI_COMMIT_VERIFY | BAPI_TRANSACTION_COMMIT 後にデータが確定すること |
| BAPI（BAPI呼出） | 更新失敗 | 異常 | BRANCH_BIZ | エラーメッセージが表示されること |
| BAPI（BAPI呼出） | 更新失敗 | 異常 | BAPI_RETURN_VERIFY | BAPI RETURN テーブルにエラーが含まれること |
| BAPI（BAPI呼出） | 更新失敗 | 異常 | ERROR_MSG | エラーメッセージ（タイプ E）が表示されること |
| BAPI（BAPI呼出） | 更新失敗 | 異常 | ERROR_LOGIC | エラー処理ロジックが正しく動作すること |

AIは仕様書の各処理ブロックの処理種別を見て、このマスタから自動的にテストパターンとテスト観点を生成します。
マスタにないパターンは生成されないため、**テスト漏れを防ぐにはこのマスタを充実させる必要があります**。

## 1. 処理種別一覧

| コード | 処理種別名 | 説明 | 典型的な使用場面 |
|--------|-----------|------|----------------|
| AUTH | 権限チェック | ユーザーの操作権限を確認する処理 | 会社コード権限、販売組織権限、プラント権限 |
| EXIST | 存在チェック | マスタデータやコードの存在を確認する処理 | 会社コード存在確認（T001）、品目マスタ確認 |
| F4HELP | F4ヘルプ | 選択画面の入力補助（検索ヘルプ）を提供する処理 | ファイル選択ダイアログ、標準検索ヘルプ |
| FILE_READ | ファイル読込 | ローカルまたはサーバーからファイルを読み込む処理 | TSV/CSV/Excel ファイルの取込 |
| FILE_WRITE | ファイル出力 | ローカルまたはサーバーにファイルを出力する処理 | TSV/CSV/Excel ファイルのダウンロード |
| VALIDATE | 入力チェック | 入力値のバリデーションを行う処理 | 必須チェック、範囲チェック、整合性チェック |
| ALV | ALV一覧表示 | ALV Grid/List でデータを一覧表示する処理 | 在庫一覧、受注一覧、ログ一覧 |
| MSG | メッセージ出力 | 処理結果のメッセージを表示する処理 | 正常終了メッセージ、エラーメッセージ |
| DB_READ | データ読込 | データベーステーブルからデータを取得する処理 | SELECT 文によるデータ取得 |
| DB_WRITE | データ書込 | データベーステーブルにデータを登録・更新・削除する処理 | INSERT/UPDATE/DELETE |
| CALC | 計算・集計 | 金額計算、数量集計などの演算処理 | 金額集計、合計算出、通貨換算 |
| CONVERT | データ変換 | データ形式の変換を行う処理 | TSV->内部テーブル変換、型変換 |
| COMMIT | コミット制御 | トランザクションの確定・取消を行う処理 | COMMIT WORK / ROLLBACK WORK |
| COMMIT_ROLLBACK | コミット制御（エイリアス） | COMMIT のエイリアス。basic_design.md で "COMMIT_ROLLBACK" を使用する場合に対応 | COMMIT WORK / ROLLBACK WORK |
| LOCK | 排他制御 | オブジェクトのロック・アンロックを行う処理 | ENQUEUE/DEQUEUE、SM12 確認 |
| LOG | ログ出力 | アプリケーションログ（SLG1）に記録する処理 | 処理ログ、エラーログ |
| PRINT | 帳票出力 | 帳票・リストの印刷出力を行う処理 | 納品書、請求書、在庫表 |
| MAIL | メール送信 | メールを送信する処理 | 通知メール、承認依頼メール |
| RFC | RFC呼出 | RFC/BAPI で外部システムと通信する処理 | 他システムとのデータ連携 |
| BAPI | BAPI呼出 | BAPI による更新処理（v2 で新規追加） | BAPI_MATERIAL_SAVEDATA 等 |
| BADI | BAdI拡張 | BAdI/Enhancement Point による標準機能拡張 | 標準プログラムへの機能追加 |
| BDC | BDCバッチ入力 | バッチ入力（BDC）でトランザクションを自動実行する処理 | マスタ一括登録、伝票一括作成 |
| IDOC | IDoc連携 | IDoc による外部システム連携処理 | EDI 連携、システム間データ転送 |
| WF | ワークフロー | SAP ワークフローの起動・制御処理 | 承認ワークフロー、通知ワークフロー |
| OTHER | その他 | 上記に分類できない処理（branches は空） | -- |

## 6. パターン追加方法

新しい処理種別やテストパターンを追加する場合:

1. `test_perspective_master.yaml` の `processing_types` セクションに新しい処理種別を追加
2. 各分岐パターン（正常/異常）に `perspectives`（テスト観点コード）と `default_expected_result`（期待結果）を定義
3. 新しいテスト観点コードを使う場合は `perspective_expected_checks` セクションにも追加
4. 必要に応じて `screen_perspectives` や `mixed_check_patterns` セクションも更新
5. このドキュメントの該当セクションも更新

**YAML 記述例（新規処理種別の追加）:**

```yaml
processing_types:
  NEW_TYPE:           # 処理種別コード
    branches:
    - pattern: 正常パターン名
      type: normal
      perspectives:
      - BRANCH_BIZ    # テスト観点コード
      default_expected_result:
      - check: no_error
    - pattern: 異常パターン名
      type: abnormal
      perspectives:
      - BRANCH_BIZ
      - ERROR_MSG
      default_expected_result:
      - check: message_type
        value: E
      - check: message_text_contains
        value: __PLACEHOLDER__

perspective_expected_checks:
  NEW_PERSPECTIVE:    # 新規テスト観点コード
    success:
      - { check: "no_error", description: "正常時の説明" }
    failure:
      - { check: "message_type", description: "異常時の説明" }
```

**check 種別一覧（default_expected_result で使用可能）:**

| check 種別 | 説明 | 主な用途 |
|-----------|------|---------|
| no_error | エラーなし確認 | 正常系全般 |
| message_type | メッセージタイプ確認（S/E/I/W） | メッセージ出力系 |
| message_text_contains | メッセージテキスト部分一致 | エラーメッセージ内容確認 |
| alv | ALV 表示確認 | ALV 表示 |
| alv_min_rows | ALV 最小行数確認 | ALV データ件数 |
| screen_number | 画面番号確認 | 画面遷移確認 |
| db_changed | DB 変更確認 | INSERT/UPDATE/DELETE/ROLLBACK |
| file_output | ファイル出力確認 | ファイル出力系 |
| log_output | ログ出力確認 | アプリケーションログ |
| print_output | 帳票出力確認 | 帳票出力 |
| idoc_status | IDoc ステータス確認 | IDoc 連携 |
| wf_status | ワークフローステータス確認 | ワークフロー |
| mail_sent | メール送信確認 | メール送信 |
| bdc_result | BDC 結果確認 | BDC バッチ入力 |
| rfc_return | RFC 戻り値確認 | RFC 呼出 |
| bapi_return | BAPI 戻り値確認 | BAPI 呼出 |
| lock_verify | ロック確認 | 排他制御 |
