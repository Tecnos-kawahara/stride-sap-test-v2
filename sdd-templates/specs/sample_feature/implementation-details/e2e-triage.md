# E2E Triage (Web-EDI受注受付)

## Purpose
E2E失敗時に原因を分類し、Spec/Plan/Tasksへ還流する。

## Categories
| Category | Definition | Action |
|---|---|---|
| product_bug | プロダクトのバグ | 修正タスク（T-*）作成 |
| spec_gap | 仕様の曖昧さ/不足 | Spec AC を更新 |
| test_bug | テストコードの不具合 | テスト修正 |
| flake | 一時的な不安定 | 安定化（待機/データ/再試験） |

## Required Artifacts
- HTML report
- Trace
- Screenshot
- Video

## Feedback Targets
- Spec / Plan / Tasks
